Word add-in

word-md

Paste Markdown into Word with proper styles applied — not as pasted-from-web junk, as native authored content.

Download manifest.xml GitHub →

What it does

word-md is a task pane add-in. Paste any Markdown — or load a .md file — and click Insert. It walks the token tree and applies named Word styles to every block, so the result behaves like content you typed with the ribbon: change the document's Heading 2 definition and every H2 from your markdown updates with it.

  • Heading 1–6 → Word heading styles
  • Bullet and numbered lists
  • Fenced code blocks
  • Inline bold, italic, strikethrough
  • Inline code (Consolas + highlight)
  • Hyperlinks
  • Pipe tables → real Word tables
  • Blockquotes
  • YAML frontmatter stripping
  • Typographic quote conversion

Install

No store account or admin access needed. Sideload the manifest directly in Word. The manifest URL (you can paste this directly in the upload dialog on some platforms). For local development and shared folder setup, see sideload.md.

https://m8urnett.github.io/word-md/manifest.xml

Security Warning: The URL above is for demonstration purposes. For production use, host these files on your own trusted infrastructure. This ensures you maintain control over the scripts executing within your documents.

Word for Windows or Mac
  1. Open Word. Go to Insert → Get Add-ins.
  2. In the dialog: My Add-ins tab → Manage My Add-ins → Upload My Add-in.
  3. Browse to manifest.xml (downloaded above) and click Upload.
  4. The word-md group appears on the Home tab. Click Paste to open the pane.
Word for the web
  1. Open a document at word.office.com.
  2. Home → Add-ins → More Add-ins → Upload My Add-in.
  3. Browse to manifest.xml and upload.
Organisation-wide (Microsoft 365 admin)
  1. Go to the Microsoft 365 admin center → Settings → Integrated apps.
  2. Upload the manifest and assign to users. No review or AppSource account required.

Privacy

word-md processes no Markdown content outside your local Word session. The task pane loads office.js from Microsoft, which is required by Office add-ins, and loads the Markdown parser from this same hosted package. No pasted content is sent to the project maintainer or to a project backend.
